# CXL Marvell/XConn Reference Sweep

Date: 2026-05-13
Scope: vendor and standards references for Project Coherent Storage CXL governance, with emphasis on Marvell, XConn, and OCP Composable Memory Systems (CMS).

## Executive finding

The Marvell/XConn public material supports treating CXL as a first-class, governed T1/T1.5 memory tier for Project Coherent Storage, especially for warm KV/prefix staging, large shared capacity pools, and memory-pressure relief. It does **not** support making CXL an unqualified replacement for local DRAM, durable NAND/OpenZFS media, or GPU/DPU-local hot paths. The important update is nuance around switching: hidden or incidental PCIe switching remains disallowed for latency-sensitive paths, but explicit CXL switch fabrics are now a vendor-roadmap reality and should be admitted for pooled/warm roles only after topology, latency, bandwidth, ownership, telemetry, and failure-domain qualification.

## ADR deltas recommended from the sweep

1. **Qualified CXL switching is allowed, hidden switching is not.** ADR-015 should retain the prohibition on hidden PCIe switching, auto-bifurcation, oversubscription, and unmeasured switch hops for latency-sensitive paths. It should also explicitly allow purpose-built CXL switch fabrics for governed T1.5 pooled/warm memory when the switch, fabric manager, ownership/fencing, link speed/width, hop count, and telemetry are qualified.
2. **CXL roadmap is current, not speculative.** Marvell/XConn public materials show CXL 2.0 switch products in production and CXL 3.1/PCIe 6.x switch roadmaps/sampling before the 2026 ADR date. The ADR should avoid calling CXL pooling experimental as a category; only our unqualified deployment modes are experimental.
3. **Use OCP CMS as the governance vocabulary.** Direct-attached, multi-headed, fabric, network, and alternate-transport classes map well to Project Coherent Storage hardware decision trees and failure domains.
4. **Add CXL fabric-manager and hotness telemetry to scheduler admission.** Coherence-CE rollup should include not only capacity/latency but also pool ownership epoch, fabric-manager health, CFM API freshness, hotness/migration state, compression ratio where applicable, and switch/fabric error state.
5. **Preserve vLLM isolation.** None of these sources changes the API contract: vLLM adapters still talk only to Coherence-CE and receive Coherence-owned placement/admission results, never CXL/OpenZFS/DPU/RDMA details.

## Gaps / not found in public first pass

- Public XConn product briefs/datasheets with register-level management details were not found; public materials are mostly press releases, demos, and media pages.
- Public Marvell/XConn performance tuning guides for production CXL memory pools were not found. The ADR should therefore require local qualification: p50/p95/p99 latency, bandwidth under concurrent GPU/DPU/NVMe load, compression-ratio sensitivity, RAS fault injection, pool ownership/fencing, failover, and scheduler admission impact.
- Public OCP CMS documents are strong for architecture/API vocabulary, but Project Coherent Storage still needs local mapping from CMS/CFM concepts into Coherence-CE admission summaries.
